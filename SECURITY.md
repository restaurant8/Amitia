# Security Policy / 安全政策

## 支持版本

当前支持的版本范围：当前主分支和当前公开发布版本。

## 报告安全漏洞

请不要通过公开 Issue 披露敏感漏洞。安全漏洞报告请发送至：

3151508592@qq.com

如果尚未配置安全联系邮箱，请通过项目维护者已公开的私有渠道联系。

报告建议包含：

- 受影响版本
- 漏洞描述
- 复现步骤
- 潜在影响
- 建议修复

请不要在报告中提交用户聊天记录、Token、API Key、微信凭证、QQ 凭证、数据库备份或其他敏感数据。必要时请先脱敏。

许可证问题、商业授权问题和安全漏洞属于不同事项，请分别使用对应联系渠道。

## English Summary

Do not disclose sensitive vulnerabilities through public issues. Security reports should include affected versions, vulnerability description, reproduction steps, potential impact, and suggested remediation. Do not include chat logs, tokens, API keys, WeChat credentials, QQ credentials, database backups, or other sensitive data unless properly redacted.
